## 改修1：祝日・休日カレンダーマスター

### ステータス：**実装可**

### 背景・目的
- 日本の祝日・会社カレンダーをあらかじめ登録しておきたい
- 現在の手動トグル（`state.holidays`）は廃止し、すべてマスターで管理する

---

### 確定仕様

#### 表示範囲
- 現状のまま（2026-01-01 〜 2026-12-31）。範囲拡張は今回対象外。
- 祝日マスターのデータは複数年分（2025〜2027）をハードコードしておく（将来の範囲拡張に備えて）。

#### カレンダー上の表示
- 祝日は**日曜日と同じスタイル**：グレー背景 ＋ 日付数字を赤色
- カレンダーヘッダーに祝日名のテキストは表示しない（色だけで区別）

#### 管理パネル（UI）
- ツールバーに **「📅 休日」ボタン** を追加
- パネル内に祝日名・日付・種別を一覧表示
- 国民の祝日は参照のみ（編集不可）
- 会社休日・カスタム休日は追加・削除可能
  - 入力項目：日付 / 名称 / 種別（会社休日 / その他）

#### 手動トグル（廃止）
- `state.holidays` 配列を廃止
- 日付ヘッダーのクリックによる平日↔休日トグルも削除
- `isGrey` 関数を `holidayMaster` のみ参照に変更

---

### データ構造

#### `state` の変更
```js
// 廃止
state.holidays  // ← 削除

// 追加
state.holidayMaster = {
  '2026-01-01': { name: '元日',     type: 'public' },
  '2026-01-12': { name: '成人の日', type: 'public' },
  '2026-07-20': { name: '海の日',   type: 'company' },
  // ...
}
// type: 'public'（国民の祝日）/ 'company'（会社休日）/ 'custom'（その他）
```

#### `isGrey` 関数の変更
```js
// 変更前
const isGrey = s => isWeekend(s) || state.holidays.includes(s);

// 変更後
const isGrey = s => isWeekend(s) || (s in (state.holidayMaster || {}));
```

#### 日付ヘッダーの色制御（変更後）
```js
// 祝日 or 日曜 → 赤文字
// 土曜 → 青文字
// 平日 → 通常色
const isHoliday = s => s in (state.holidayMaster || {});
const dayColor  = s => isHoliday(s) || parse(s).getDay()===0 ? '#d32f2f'
                     : parse(s).getDay()===6 ? '#1565c0' : '#37474f';
```

## 改修2：空セルへの入力をダブルクリックに変更

### ステータス：**実装可**

### 背景・目的
- 現在：空セル**シングルクリック**でインライン入力フィールドが開く
- 問題：カレンダーをドラッグスクロール中に意図せず入力モードになる
- 変更：空セルへの入力開始を**ダブルクリック（ダブルタップ）**に変更する

---

### 確定仕様

| 操作 | 変更前 | 変更後 |
|---|---|---|
| 空セル シングルクリック | インライン入力が開く | **何もしない**（手のマークのまま） |
| 空セル ダブルクリック | — | **インライン入力が開く**（新） |
| Shift+ドラッグ 範囲選択 | 離したときインライン入力 | 変更なし |
| タスク シングルクリック | タスク選択 | 変更なし |
| タスク ダブルクリック | 編集ポップ | 変更なし |
| メモ欄 ダブルクリック | メモ編集 | 変更なし |
| ルーティン行 ダブルクリック | セル入力 | 変更なし |

---

### 変更箇所（`app.js` 1箇所のみ）

`window.addEventListener('pointerup', ...)` 内、約500行目：

```js
// 変更前
} else if(startCell && startCell.di>=0 && startCell.di<days.length){
  openInline(startCell.di, startCell.di, startCell.ln);
}

// 変更後
} else if(startCell && startCell.di>=0 && startCell.di<days.length){
  const cid = 'cell:' + startCell.di + ':' + startCell.ln;
  const now = Date.now();
  if(lastTapId === cid && now - lastTapTime < DBL_MS){
    lastTapId = null; lastTapTime = 0;
    openInline(startCell.di, startCell.di, startCell.ln);
  } else {
    lastTapId = cid; lastTapTime = now;
  }
}
```

- `DBL_MS=320ms`：既存のダブルクリック検出と同じ値を使用
- iPhone（ダブルタップ）も同じロジックで動作する

---

### 実装対象ファイル
- `app.js`：上記1箇所のみ

## 改修3：メモ欄を5行に増やし、1行を細くする

### ステータス：**実装可**

### 背景・目的
- 現在：メモ欄は2行（`dayNotes` / `dayNotes2`）、1行 = 100px
- 変更：5行に増やし、1行を細くして使いやすくする

---

| 項目 | 変更前 | 変更後 |
|---|---|---|
| 行数 | 2行 | **5行** |
| 1行の高さ（`NOTE_H`） | 100px | **67px**（約2/3） |

## 改修4：メモ欄のD&D移動 ＋ タスクD&Dに競合チェック追加

### ステータス：**実装可**

### 背景・目的
- メモ欄にD&D移動機能を追加する
- 同時に、タスクD&Dにも競合チェックを追加する（現在は競合無視で上書き）

---

### 確定仕様：共通ルール（タスク・メモ共通）

| ドロップ先の状態 | 動作 |
|---|---|
| **空き** | 移動成功。元の場所は空になる |
| **すでに内容あり** | 移動せず元の位置に戻る。ポップアップで理由を表示 |
| **メモ欄以外・タスク欄以外** | キャンセル。元の位置に戻る（ポップアップなし） |

---

### タスクD&Dの変更

#### 「競合あり」の定義
- ドロップ先の **日付 ＋ レーン** にすでに別のタスク（`deleted:false`）が存在する場合

#### 変更前の動作
```js
// 競合チェックなし。そのまま上書き移動
td.t.start = days[td.previewDi];
td.t.lane  = td.previewLn;
save(); render();
```

#### 変更後の動作
```js
const conflict = state.tasks.some(t =>
  !t.deleted && t.id !== td.t.id &&
  t.lane === td.previewLn &&
  t.start === days[td.previewDi]
);
if(conflict){
  showAlert('このセルにはすでにタスクが入っています');
  render(); // 元の位置に戻す（taskDrag破棄済みなので再描画だけでOK）
} else {
  td.t.start = days[td.previewDi];
  td.t.end   = dur>1 ? days[td.previewDi+dur-1] : '';
  td.t.lane  = td.previewLn;
  stamp(td.t); selectedId=td.t.id; save(); render();
}
```

---

### メモD&Dの仕様

#### ドラッグ開始条件
- 入力済み（空でない）メモセルの `pointerdown` でドラッグ開始
- 空セルはドラッグ開始しない

#### ドラッグ中の視覚フィードバック
- ドラッグ中のセルを半透明（タスクの `dragging` クラスと同様）
- ホバー中の移動先セルをハイライト

#### ドロップ処理
```js
const targetVal = state[targetNotesKey][targetDate] || '';
if(targetVal !== ''){
  showAlert('このセルにはすでにコメントが入力されています');
  render(); // 元の位置に戻す
} else {
  state[targetNotesKey][targetDate] = state[sourceNotesKey][sourceDate];
  state[sourceNotesKey][sourceDate] = '';
  save(); render();
}
```

- 移動先は**日付・行を問わない**（メモ①→別の日のメモ③ なども可）

---

### 実装対象ファイル
- `app.js`：
  - タスクD&Dのドロップ処理に競合チェック追加
  - メモセルのドラッグ検出（`pointerdown` / `pointermove` / `pointerup`）
  - ドラッグ中のハイライト処理
  - ドロップ先の判定（日付・メモ行番号の特定）
  - 移動処理 / ポップアップ表示
